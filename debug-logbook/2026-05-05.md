## Open Follow-Ups

- Lock down unauthenticated callable/admin-like Firebase functions, especially functions that inspect registrations, create test registrations, or repair/sync enrollment counters.
- Remove or disable the hardcoded ngrok/simple booking webhook path and move webhook configuration to safe production config/secrets.
- Fix Stripe fallback behavior where unset/free workshops may default to a HKD 500 charge.
- Add ownership/auth checks to `confirmPaymentUpload` to prevent registration ID abuse.
- Restrict `requestFreeMaterialDownload` so client-provided `pdfUrl` cannot be used for SSRF or quota abuse.
- Add a safe allowlist to `storageProxy` so it cannot bypass Storage Rules for arbitrary object paths.
- Reconsider anonymous `registrations/{id}` reads in Firestore rules.
- Make `publicWorkshopEvents` return only intended public fields and default to visible-only behavior.

- Remove misleading `1堂` display for non-continuous multi-session public workshop cards.
